# Sprint 19 - 2026-05-05

## Modifiche Create
- Formattazione dinamica e salvataggio normalizzato dei form in `components/MasterDataModal.tsx`.
- **REGIONE / STATO**:
  - Il sistema permette l'inserimento sia liberi sia codificati. 
  - Al momento del salvataggio: se la parola è 'Italia' o 'ITA' (senza distinzione di case) o qualsiasi stringa a 3 caratteri esatti considerabile codice di stato, questa viene archiviata convertita integralmente in maiuscolo (Es. "Italia" -> "ITA").
  - Testo con più o meno di tre lettere ("Puglia") viene preservato considerandolo regione.
- **PROVINCIA**:
  - Implementato un vincolo *Real-time* nel modulo di compilazione alla digitazione.
  - La stringa viene forzata maiuscola in ogni momento, rimossi i caratteri non alfabetici e limitato rigidamente alla lunghezza massima di 3 lettere ("BA" -> "BA", "BAR" -> "BAR"). La digitazione involontaria di nomi estesi non è processata.
- **NUMERI DI TELEFONO**:
  - I campi per telefono (`Telefono Principale`, `Amministrazione`, ecc.) ricevono un sistema di pattern *Real-time*.
  - Condizione di accensione: se la regione omessa, maggiore di 3 lettere (Italiana) o corrispondente a 'ITA'.
  - Pre-compilazione prefisso: se la sequenza parte con `0` (fisso) o `3` (cellulare) in tempo reale l'app inserisce prefisso `+39` in testa (+ spazio) e raggruppa il telefono con spazi ogni segmento 3-4 digit.

## Risposte
**Firebase Deploy Functions:**
No, per questa applicazione non sono previste e non sussiste alcun comando CLI `firebase deploy --only functions`. La logica cloud è interamente appoggiata a Firebase Storage, Database Firestore Client-side e Firebase Auth. Tutte le manipolazioni e verifiche occorrono nativamente via l'SDK all'interno del progetto stesso nel browser tramite Vite/Rollup. Non c'è alcun backend *Node.js* da far girare nelle Google Cloud Functions associato a questa architettura.

## Funzionamento e Test
Le formattazioni sono estensive per tutte le anagrafiche Clienti, Articoli e Fornitori trattati dalla Modale Master Data. 

## Effetti sul sistema
Maggiore pulizia e compliance dei dati d'origine. L'uso dei dati filtrati previene incoerenza logica. I cambiamenti del codice sono limitati al controllo input in runtime. Nessuna rottura attesa per letture pregresse poiché nessuna struttura e schema database è stato alterato.
