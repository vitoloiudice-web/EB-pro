# SPRINT 19 - 21/04/2026

## COSA È STATO CREATO E A COSA SERVE
1.  **Bugfix PWA "Costruttore/Schermo Bianco" in Vercel**
    *   **Perché:** Il precedente *Service Worker* (sw.js manuale) non pre-cachava correttamente l'asset manifest. Inoltre l'importante bundling generato da librerie di peso elevato come *SheetJS (xlsx)* generava un singolo file JS oltre i 2MB (2.97MB nel log di build), che aggirava la cache di default costringendo il loader SPA a restiture il Javascript come file sorgente crudo visualizzato a schermo (Copyright Microsoft Corporation, etc...).

## COME È FATTO E COME FUNZIONA
*   **Vite PWA Plugin (`vite-plugin-pwa`):** Cancellato il file artigianale `sw.js` da `/public` (non ottimizzato e deprecato) in favore del plugin ufficiale PWA di ViteJS.
*   **Gestione Chunking (Workbox):** Incrementato il massimale di pre-cache `maximumFileSizeToCacheInBytes` a 5MB, permettendo all'applicazione di salvarsi interamente in memoria locale sul dispositivo mobile superando lo sbarramento.
*   **Manifest Dymamic Injection:** Rimossa la dichiarazione `<link rel="manifest">` hardcore nel index e istruito Vite a generarla dinamicamente agganciata alla versione corretta di hashing della build per PWA. E cancellato il `manifest.json` statico vecchio per non fare duplicato con la cache.

## EFFETTI E PROPAGAZIONE
*   **UI/UX:** Installando da Safari/Chrome mobile, adesso il sistema operativo è in grado di rintracciare e montare il file entry point corretto (Index.html -> React -> Vite) al posto di iniettare codice puro sul rendering object.
*   **Deployment:** La migrazione al plugin PWA di Vite garantisce che tutte le dipendenze siano compresse con l'app per permetterne l'eventuale uso offline limitato alle sole query memorizzate.

## REGOLE DA SEGUIRE ORA
Effettuare un nuovo **commit GitHub** per forzare la Vercel Build. Disinstallare l'app "rotta" dallo smartphone, cancellare cache di sistema, e procedere ad una nuova installazione in Homescreen usando il nuovo deployment di Vercel.
